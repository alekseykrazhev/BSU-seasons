Вопрос 1.

Для такой классификации (группировки) пользователей стоит сделать некоторые первичные шаги (исследования). Приведу пример-схему для такого исследования.
Объекты исследования: пользователи (социально-демографические данные; цели, ожидания, мотивация; способности; задачи и сценарии; контекст использования),технологии (ресурсы, ограничения),заказчик (критерии успешности).
К направлениям исследования можно отнести следующие:
1) изучение предметной области, выявление потребностей всех заинтересованных лиц и в частности заказчика
2) изучение пользователей и выявление их потребностей

Далее будет следовать анализ собранных данных для группировки. Цель анализа - создание профилей групп, группировка пользователей, изучение контекстных сценариев. Задачи анализа - обработка и изучение результатов проведенного анализа, группировка пользователей по этим результатам.

У групп пользователей есть свои ключевые характеристики. Приведу примеры создания групп (разных видов групп).
Группы могут быть составлены: по поведению, по потребностям, по контексту, по ценностям пользователей, по знаниям, по месту проживания/этнической принадлежности.

У конечного заказчика есть свои приоритеты для групп пользователей. Таким образом, в первую очередь он обращает внимание на уже существующих пользователей, затем на привлечение новых пользователей, затем пользователи, кто не остается на приложении заказчика надолго. Перечислю еще несколько таких групп в соответствии с приоритетами:
- неопытные пользователи
- группа пользователей, которая приносит наибольший доход
- самая многочисленная группа и так далее

Исследования могут быть следующие: интервью конечных пользователей, экспертов в области, обзор литературы, аудит прототипа.
Данные можно получать у конкурентов, из данных от заказчика.

Диаграмма сходства применяется для анализа данных, для соответствующего создания групп пользователей конечного продукта, для анализа полученных данных и оценки юзабилити. После такого исследования мы получаем тематически сгуппированные наблюдения.

Для выполнения сортировки по шкалам необходимо: выявить поведенческие переменные, для переменных сопоставить их с респондентами, выявить важные шаблоны, синтезировать связи "характеристики - цели", оценить полноту сделанной работы, и перейти к созданию персонажей.

Вопрос 2. Концептуальные принципы (Проектирование для пользователей с разным уровнем подготовки и поддержка состояния потока)

Проектирование для пользователей с разным уровнем подготовки и поддержка состояния потока являются двумя важными концептуальными принципами, которые помогают создавать удобные и интуитивно понятные пользовательские интерфейсы. Ниже приведен каждый из этих принципов более подробно:

Проектирование для пользователей с разным уровнем подготовки:
Этот принцип заключается в создании пользовательского интерфейса, который будет понятен и доступен для пользователей с различным уровнем подготовки и опытом работы с компьютерами. Вот некоторые рекомендации, которые помогут вам следовать этому принципу:

- Используйте понятные и легко узнаваемые символы и значки.
- Предоставьте четкие инструкции и подсказки для выполнения задач.
- Обеспечьте логическую и последовательную структуру интерфейса.
- Предоставьте различные уровни сложности или режимы работы, чтобы удовлетворить разные потребности пользователей.

Поддержка состояния потока:
Этот принцип связан с сохранением и восстановлением состояния пользовательского интерфейса, чтобы пользователи могли легко вернуться к предыдущим задачам или продолжить работу после перерыва. Вот некоторые способы поддержки состояния потока:

- Сохраняйте и восстанавливайте введенные данные или прогресс пользователя между сеансами работы.
- Предоставьте функцию отмены и повтора, чтобы пользователи могли легко откатить или повторить свои действия.
- Обеспечьте навигацию с помощью истории или вкладок, чтобы пользователи могли легко перемещаться по предыдущим состояниям интерфейса.
- Разработайте удобные средства сохранения и загрузки проектов или сеансов работы.

Следование этим принципам поможет вам достичь большего количества пользователей с разным уровнем подготовки. Важно соблюдать эти принципы для получения максимально хорошего результата для вашего проекта/приложения.

Вопрос 3. Синтаксис html

HTML - hyper text markup language. стандартный язык разметки веб страниц. это теговый язык разметки документов.

Весь html документ состоит из соответствующих тегов, которые открываются и закрываются (не всегда). Например <br> закрывать не нужно, он одиночный (тег для переноса строки).

У тегов могут быть атрибуты. Они указываются в первом(открывающем) теге. Например, <a href="ссылка">ТУТ ЕСТЬ ГИПЕРССЫЛКА</a>. Также в таких тегах может быть и атрибуты для текста, описаны css стили или js скрипты и многое другое в зависимости от самого тега и его предназначения.

Комментарии в html выглядят следующим образом: <!-- комментарии! -->
Для примера приведу несколько популярных html тегов: <input> - для создания формы ввода, и <img> для отображения изображения.

Разметка в html может быть физическая и логическая.
Физическая разметка заключается в использовании тегов для создания визуального отображения страницы документа. Т.е. оно заклычается в инструкциях отображения страницы в браузере (внешнего вида). Тут используются такие теги, как <big>, <sup>, <sub>, <p>

Логическое форматирование - это больше относится к структуре, т.е. к логической разметке документа. Тут также используются теги, но они определяют степень важности отображения элементов, его отношения к типу данных и тому подобное.
Сюда относятся такие теги, как <code>, <del>, <em>, <strong>.
